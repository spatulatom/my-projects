Manage Spotify is a React.js web app that connects to Spotify API via 'Implicit Gran Flow'. The implicit grant flow is carried out on the client side and it does not involve secret keys. Thus, no need for any server-side code to use it. Access tokens issued are short-lived with no refresh token to extend them when they expire. The user on login gets onAuth window and can chosse the way of loging into their own account throug this app. The Spotify API allows an array of functionalities. I am only exploring a few of them for example a user can see all their playlists names, see particular playlist content, add song to the existing playlists, add entirly new playlist to their account.

The app is build with React.js cwith class based componets and therfore with of lifecycle methods as well. It was intersting to see that older version of React in action. I admit though that with introduction of Hooks (React v.16.8.0) and functional components getting 'full' status thing are made much straigtfoward. Still It was a great exercise!

MAIN PROJECT

My main project is a MERN web app called 'eventsbook'. Its functionality is somewhere between social platforms like Facebook and Instagram. I provides create account/login interface for the potential user, and password change (you can also create account with dummy email address but you won't be able to change your password). Then new user can create an event, and that is 'a new photo': picture with the description and location, or 'a new post': only description and location (part of the decription can be an url to another http address). Users can modify only their own added events (the can not modify events added by other Users) by clcking button 'CHANGE', or remove them by clickin button 'DELETE'. For every events every user can see their location on the map by clickin button 'MAP', add comment and like (or remove like) an event.

Frontend of it is up to date React.js (React 18), delpoyed on firebase; backend is Node, Express.js and Mongoose connected to MongoDb, deployed on Heroku. Front and backend are both using Google Map Platform API for the 'location' functionality (mentioned earlier), backend additionaly is using Amazon aws services API for permanentlu storing uploaded photos, SendGrid API for changing password/notification email upon creating account. The app is not using Redux since React own hook like useReducer for complex state and useContext for sharing certain state across the whole app are sufficient. There are custom hooks built and that is form-hook.js for managing form data, http-hook.js for all fetcjing requests and auth-hook.js for managing backend login token throught the rerender cycles. As for the CSS styling I follow (try anyways) BEM convention.

There is currently a few dummy users account created and some of them shared their events, liked other events and made comments. You welcome to create an account for yourself (it's only a minute) and explore the app for yourself.
