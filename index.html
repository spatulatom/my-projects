MAIN PROJECT

My main project is a MERN app called 'eventsbook', it is delopyed on Firebase, while the backend server is running on Heroku.

See the deployed app on firebase .

See eventsbook-frontend on github .

See eventsbook-backend on github .


Its functionality is somewhere between social platforms like Facebook and Instagram (to state it very boldly:)). It provides a create account/login interface for the potential user, and a password change (the new account can be also created with a dummy email address but in that case, there won't be a possibility for a password change). Then the new user can create an event, and that is 'a new photo': a picture with a description and a location, or 'a new post': only a description and a location (part of a description can be an URL to another HTTP address). Users can modify only their own added events (they can not modify events added by other Users) by clicking the button 'CHANGE', or by clicking the button 'DELETE'. Users can comment on all events, like them or remove their likes, and see the event's location on google maps.

The frontend of this app is up-to-date React.js (React 18), deployed on firebase; the backend is Node, Express.js and Mongoose connected to MongoDb, deployed on Heroku. The frontend and the backend are both using Google Map Platform API for the 'location' functionality (mentioned earlier), the backend additionally is using Amazon AWS services API for permanently storing uploaded photos, SendGrid API for changing password/notification email upon creating an account. The app is not using Redux since React own hooks like useReducer for managing complex state and useContext for sharing certain state data across the whole app seem sufficient. There are custom hooks built and that is form-hook.js for managing form data, http-hook.js for all fetching requests and auth-hook.js for managing backend login token through the rerender cycles. As for the CSS styling, I follow (try anyway) the BEM convention.

There are currently a few dummy user accounts created and some of them users shared their events, liked other events and made some comments under certain events (all photos currently used in the app are free to use photos from https://www.pexels.com/). You are welcome to create a new account or login with email: user1@gmail.com, password: user01, and explore the app for yourself.

I find this app to be a very exciting project with a variety of possible ways for further development. Currently, a store section is being added that will provide users with an option to add a product (related to the event) to a basket and proceed to the (dummy) PayPal payment.
