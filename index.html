MAIN PROJECT

My main project is a MERN web app called 'eventsbook'. Its functionality is somewhere between social platforms like Facebook and Instagram. I provides create account/login interface for the potential user, and password change (you can also create account with dummy email address but you won't be able to change your password). Then new user can create an event, and that is 'a new photo': picture with the description and location, or 'a new post': only description and location (part of the decription can be an url to another http address). Users can modify only their own added events (the can not modify events added by other Users) by clcking button 'CHANGE', or remove them by clickin button 'DELETE'. For every events every user can see their location on the map by clickin button 'MAP', add comment and like (or remove like) an event.

Frontend of it is up to date React.js (React 18), delpoyed on firebase; backend is Node, Express.js and Mongoose connected to MongoDb, deployed on Heroku. Front and backend are both using Google Map Platform API for the 'location' functionality (mentioned earlier), backend additionaly is using Amazon aws services API for permanentlu storing uploaded photos, SendGrid API for changing password/notification email upon creating account. The app is not using Redux since React own hook like useReducer for complex state and useContext for sharing certain state across the whole app are sufficient. There are custom hooks built and that is form-hook.js for managing form data, http-hook.js for all fetcjing requests and auth-hook.js for managing backend login token throught the rerender cycles.

See the deployed app on firebase .

See eventsbook-frontend on github .

See eventsbook-backend on github .
